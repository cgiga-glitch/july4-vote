# Vicky Oh 랙 홍보 지원 스케줄 설문 웹앱

직원들이 캘린더에서 도와줄 수 있는 시간을 드래그로 선택해 제출하고,
모든 직원이 결과를 대시보드(히트맵 + 인원별 누적시간)로 볼 수 있는 웹앱입니다.

구조: **GitHub Pages(프론트엔드, 정적 사이트)** + **Google Apps Script(백엔드 API, Google Sheets 연동)**
별도 서버/비용 없이 무료로 운영됩니다.

---

## 1. Google Sheet 만들기 (직접 생성)

새 스프레드시트를 만들고 시트 이름을 **`Responses`** 로 지정한 뒤, 1행에 아래 헤더를 정확히 입력하세요.

| 열 | 헤더 (1행) | 설명 | 예시 |
|---|---|---|---|
| A | Timestamp | 제출 시각 (자동 기록) | 2026-06-10 09:12:33 |
| B | Name | 직원 이름 | 홍길동 |
| C | Date | 지원 날짜 | 2026-06-14 |
| D | SlotStart | 슬롯 시작 시각 | 13:00 |
| E | SlotEnd | 슬롯 종료 시각 | 13:30 |
| F | Hours | 슬롯당 시간(항상 0.5) | 0.5 |

> 한 사람이 여러 시간대(30분 단위)를 선택하면 **행이 여러 개** 생성됩니다.
> 예: 홍길동이 13:00~14:00을 선택하면 2행(13:00-13:30 / 13:30-14:00)이 생깁니다.
> 나머지 집계(대시보드용 인원 수, 인원별 총 시간)는 Apps Script가 자동으로 계산하므로
> **시트에 별도의 요약 탭을 만들 필요는 없습니다.**

**중요**: D열(SlotStart), E열(SlotEnd)은 구글시트가 "13:00" 같은 값을 시간 형식으로
자동 변환할 수 있습니다. 이미 코드에서 이를 보정하지만, 더 안전하게 하려면
D:E 열 전체를 선택 → 서식 > 숫자 > 일반 텍스트로 지정해 두세요.

## 2. Apps Script 백엔드 연결

1. 방금 만든 스프레드시트에서 **확장 프로그램 > Apps Script** 클릭
2. `apps-script/Code.gs` 파일의 내용을 그대로 붙여넣기
3. 상단 **배포 > 새 배포**
   - 유형: **웹 앱**
   - 실행 사용자: **나(Me)**
   - 액세스 권한: **전체(Anyone)** — 직원들이 로그인 없이 이름만 입력해 제출하므로 필수
4. 배포 후 발급되는 URL(`https://script.google.com/macros/s/.../exec`)을 복사

## 3. 프론트엔드에 URL 연결

`config.js` 파일을 열어 아래 줄의 URL을 3번에서 복사한 URL로 교체합니다.

```js
const APPS_SCRIPT_URL = "https://script.google.com/macros/s/여기에_배포된_URL을_붙여넣으세요/exec";
```

## 4. GitHub Pages로 배포

1. GitHub에 새 저장소 생성 (예: `vicky-oh-schedule`)
2. 이 폴더의 파일(`index.html`, `style.css`, `app.js`, `config.js`)을 저장소 루트에 업로드
   (`apps-script/Code.gs`는 Google Apps Script 편집기에만 있으면 되고, 저장소에는 참고용으로만 두면 됩니다)
3. 저장소 **Settings > Pages** 에서 Branch를 `main` / 폴더 `/(root)` 로 지정 후 저장
4. 몇 분 후 `https://<github계정>.github.io/vicky-oh-schedule/` 로 접속 가능

## 5. 사용 방법

- **설문 참여 탭**: 이름 입력 → 캘린더에서 마우스로 드래그해 지원 가능한 시간(연두색) 선택
  → 선택한 시간은 금색으로 바뀌며 **선택한 사람의 이름이 슬롯 안에 표시됨** → 제출하기
  - **한 슬롯(30분)은 한 사람만 선택할 수 있습니다.** 이미 다른 사람이 선택한 시간은
    회색으로 잠기고 그 사람 이름이 표시되어 다른 사람은 선택할 수 없습니다.
  - 이름을 입력한 뒤 캘린더 밖을 클릭하면(포커스 아웃) 이전에 저장해둔 내 시간이
    자동으로 불러와져 선택 상태로 표시됩니다 → 그대로 두거나 드래그해서 수정 후 다시 제출하면 됩니다.
  - 다른 사람이 방금 선점해 저장에 실패한 시간이 있으면 제출 후 안내 메시지로 알려줍니다.
- **대시보드 탭**: 전체 예약 캘린더에 각 슬롯을 선택한 사람 이름이 바로 표시됩니다.
  우측 표에는 인원별 누적 지원 시간이 표시됩니다. 새로고침 버튼으로 최신 데이터를 다시 불러올 수 있습니다.

## 지원 가능 시간대 (코드에 이미 반영됨)

| 날짜 | 시간 |
|---|---|
| 6/14 (일) | 13:00 ~ 17:00 |
| 6/15 (월) | 09:00 ~ 12:00, 13:00 ~ 17:00 |
| 6/16 (화) | 09:00 ~ 12:00 |

시간대를 변경해야 하면 `app.js` 상단의 `DAYS` 배열만 수정하면 됩니다.
